📐 Always Check Normality First
Why Normality Matters

Most parametric statistical tests (t-tests, ANOVA, regression) assume the data follows a normal distribution. If violated, results can be misleading. In beverage manufacturing, most continuous measurements (fill volume, Brix, pH, turbidity, dissolved oxygen) are approximately normal when the process is in control. Sensory scores, counts, and proportions are often NOT normal.

Anderson-Darling test: p > 0.05 → data is normal (fail to reject H₀) Shapiro-Wilk test: p > 0.05 → data is normal Rule of thumb: skewness in (-1, +1) and kurtosis in (-2, +2) → approximately normal

Process Capability
Cp, Cpk, Pp, Ppk — measuring how well a process fits within specification limits. The most important indices for a Six Sigma practitioner.
📐 The Four Core Indices — Formulas & Meaning
Cp = (USL − LSL) / (6σ̂) ← potential capability (spread only) Cpk = min[ (USL−μ) / 3σ̂, (μ−LSL) / 3σ̂ ] ← actual capability (centring + spread) Pp = (USL − LSL) / (6s) ← overall performance (long-term) Ppk = min[ (USL−μ) / 3s, (μ−LSL) / 3s ] ← overall performance (centring + long-term) σ̂ = R̄/d₂ or S̄/c₄ (within-subgroup, short-term estimate) s = sample standard deviation of all data (long-term)
🔑
Key rule: Always look at Cpk, not Cp. Cp tells you if the spread could fit — Cpk tells you if it actually does fit given where the mean sits. A high Cp with low Cpk means the process is too far off-centre.
📊
Cp vs Cpk: If Cp = Cpk → process is perfectly centred. If Cpk < Cp → process is off-centre. The bigger the gap, the more the mean needs to shift.
⚠️
Cp/Cpk vs Pp/Ppk: Use Cp/Cpk to evaluate the process in the short term (are the machines capable when stable?). Use Pp/Ppk to evaluate all sources of variation including shift-to-shift, batch-to-batch, operator — the real-world performance.

🍶 Step-by-Step: Calculating Cpk on a Beverage Line
🏭
Scenario: You are a Black Belt at a carbonated soft drink plant. The specification for fill volume is 500 mL ±4 mL (LSL = 496 mL, USL = 504 mL). You collect 25 subgroups of n=5 samples each from the filler during one 8-hour shift.
  1. Collect data: Record 25 subgroups × 5 samples. Your data file has columns: Subgroup ID, Obs 1–5.
  2. Verify stability: Plot an X-bar R chart first. If there are out-of-control points, investigate and remove special causes before calculating Cpk. Capability is only meaningful for a stable process.
  3. Calculate the grand mean (μ̂): Average of all 125 observations. Result: X̄̄ = 500.6 mL.
  4. Estimate within-subgroup σ̂: Calculate R for each subgroup → average R̄. Then σ̂ = R̄ / d₂. For n=5, d₂ = 2.326. If R̄ = 1.85, then σ̂ = 1.85 / 2.326 = 0.795 mL.
  5. Calculate Cp: Cp = (504 − 496) / (6 × 0.795) = 8 / 4.77 = 1.68.
  6. Calculate Cpk: CPU = (504 − 500.6) / (3 × 0.795) = 3.4 / 2.385 = 1.43. CPL = (500.6 − 496) / 2.385 = 1.93. Cpk = min(1.43, 1.93) = 1.43.
  7. Interpret: Cp = 1.68 means the spread could fit well. Cpk = 1.43 means you are capable (≥1.33) but slightly biased toward the upper limit (mean is 0.6 mL above target). Action: re-centre fill head to reduce mean to 500.0 mL.

🔬 QC Lab Application — Cpk on Analytical Methods
🧪
Lab context: In a QC lab, you can use Cpk to assess whether your analytical method has sufficient precision to distinguish conforming from non-conforming product. This is related to — but separate from — MSA.

For a lab, calculate Cpk using your control sample results over 30+ days against your internal lab specification limits (not the product spec). This tells you: is my instrument's day-to-day repeatability good enough to make reliable product decisions?

🧪 Lab Example — Brix HPLC method:
Internal lab control target: 11.50 °Bx. Control limits: 11.40–11.60 °Bx.
30 daily QC check results: mean = 11.51, σ = 0.023 °Bx.
Cpk = min[(11.60−11.51)/(3×0.023), (11.51−11.40)/(3×0.023)]
= min[1.30, 1.59] = Cpk = 1.30 (marginal — investigate instrument calibration drift).

Control Charts
SPC (Statistical Process Control) — detecting special cause variation in real time. The cornerstone of the Control phase and ongoing production monitoring.
🗺️ Which Control Chart to Use — Decision Guide
START: What type of data are you measuring?
📏 VARIABLE (continuous) data — e.g., fill volume, Brix, pH, temperature
n = 1:I-MR Chart (Individuals & Moving Range)One reading per time period. Lab results, tank temperatures, batch Brix.
n = 2–8:X-bar R ChartSmall subgroups from a continuous process. Most common in filling lines.
n ≥ 9:X-bar S ChartLarge subgroups. Standard deviation (S) is more efficient estimator than R.
🔢 ATTRIBUTE (count/proportion) data — e.g., defective bottles, label defects
Defectives:
P-chart — proportion defective, sample size VARIES (e.g., daily batches of different size)
NP-chart — count of defectives, sample size is CONSTANT (e.g., always check 200 units)
Defects:
C-chart — count of defects per unit, inspection area CONSTANT (e.g., scratches per bottle in fixed size sample)
U-chart — defects per unit, inspection area VARIES (e.g., number of label defects per 100m of label roll)
🔑
Defectives vs Defects: A defective is a unit that fails (e.g., a bottle is defective if it fails fill volume). A defect is a specific flaw (one bottle can have multiple defects: a low fill AND a cosmetic scratch). Use P/NP for pass/fail decisions. Use C/U when counting multiple flaws per unit.
1️⃣ I-MR Chart (Individuals & Moving Range)
I chart (Individuals): UCL(I) = X̄ + 3(MR̄/d₂) ← d₂ = 1.128 for n=2 (span of 2) CL(I) = X̄ LCL(I) = X̄ − 3(MR̄/d₂) MR chart (Moving Range): UCL(MR) = D₄ × MR̄ ← D₄ = 3.267 for n=2 CL(MR) = MR̄ LCL(MR) = 0 ← always 0 for MR charts MR_i = |X_i − X_{i-1}| (each moving range = absolute difference between consecutive points)

When to use

  • One measurement per time period
  • Batch processes (one result per batch)
  • Lab results (one Brix per blend)
  • Slow processes (1 reading/shift)

How to use

  • Plot X values on I chart (top)
  • Plot MR on Moving Range chart (bottom)
  • Apply WE rules to I chart
  • MR chart: look for spikes (sudden changes)

Beverage use cases

  • Tank Brix at end of each blend
  • Pasteurisation temperature per batch
  • Alcohol % per fermentation batch
  • Daily finished product pH in lab
🧪 Lab Example — Juice Brix: QC lab measures Brix on one sample per juice blend (1 batch/day). Target: 11.5 °Bx. 20 days of data: mean X̄ = 11.52, MR̄ = 0.18. UCL(I) = 11.52 + 3×(0.18/1.128) = 11.52 + 0.479 = 11.999. LCL(I) = 11.041. On Day 17, Brix = 12.21 → above UCL → special cause → investigate syrup concentration that day.

2️⃣ X-bar R Chart
X-bar chart (process mean): UCL(X̄) = X̄̄ + A₂ × R̄ CL(X̄) = X̄̄ LCL(X̄) = X̄̄ − A₂ × R̄ R chart (process variation/range): UCL(R) = D₄ × R̄ CL(R) = R̄ LCL(R) = D₃ × R̄ Control chart constants (most common subgroup sizes): n=2: A₂=1.880, D₃=0, D₄=3.267, d₂=1.128 n=3: A₂=1.023, D₃=0, D₄=2.574, d₂=1.693 n=4: A₂=0.729, D₃=0, D₄=2.282, d₂=2.059 n=5: A₂=0.577, D₃=0, D₄=2.114, d₂=2.326 n=7: A₂=0.419, D₃=0.076, D₄=1.924, d₂=2.704
⚙️
Rational subgroups: Group samples that were produced together under the same conditions (e.g., 5 consecutive bottles). This ensures within-subgroup variation reflects only the common cause variation of the process — allowing detection of shifts between subgroups.

UCL(R) = 2.114×2.8 = 5.9 mL. Check every subgroup: at subgroup 18 R = 6.4 mL (above UCL-R) → variability spike → check for worn filler nozzle.

3️⃣ P-Chart (Proportion Defective)
p̄ = Total defectives / Total units inspected Control limits (vary with each sample size n_i): UCL(p_i) = p̄ + 3 × √(p̄(1−p̄)/n_i) LCL(p_i) = p̄ − 3 × √(p̄(1−p̄)/n_i) ← set to 0 if negative

When to use

  • Pass/fail inspection data
  • Varying batch/lot sizes
  • Monitoring defect rate over time
  • Audit findings over different lot sizes

Important rules

  • n should be large enough so np̄ ≥ 5
  • UCL/LCL change with each n_i
  • Use NP-chart if n is always constant
  • Minimum 25 subgroups to establish limits

Beverage applications

  • Daily fill volume pass/fail rate
  • Label application defect rate
  • Audit: proportion non-conforming labels per delivery
  • Bottle cosmetic defect rate

MSA / Gauge R&R
Measurement System Analysis — verifying that your measurement device and operators are reliable before trusting data for decisions. Critical in QC labs and production.
🔬 What is MSA and Why It Matters

If your measurement system is unreliable, all your Six Sigma analysis is built on sand. MSA asks: how much of the observed variation in your data is real process variation, and how much is just measurement error?

Total observed variation² = Process variation² + Measurement variation² Gauge R&R = Repeatability + Reproducibility Repeatability: same operator, same part, same gauge → multiple times EV (Equipment Variation) = how much the gauge itself varies Reproducibility: different operators, same part, same gauge AV (Appraiser Variation) = how much operators differ from each other %GRR = (GRR std dev / Total std dev) × 100 ndc = 1.41 × (Part std dev / GRR std dev) ← number of distinct categories
%GRRndcDecision
<10%≥5✅ Acceptable — use the gauge
10–30%3–4⚠️ Marginal — may be acceptable depending on application risk
>30%<3❌ Unacceptable — fix or replace measurement system

🍶 Step-by-Step: Running a Gauge R&R in a Beverage Lab
🏭
Scenario: You want to verify that your lab refractometer is suitable for measuring Brix on juice samples. The product specification is 11.5 ±0.3 °Bx. You have 2 lab analysts and will use the crossed (full factorial) Gauge R&R design.
  1. Select 10 samples that span the expected range of process variation — ideally covering from below LSL to above USL. Label them blindly (A–J). Do NOT tell operators which sample is which.
  2. Each operator measures each sample twice (2 replicates), in random order. Total: 2 operators × 10 samples × 2 replicates = 40 measurements. Record in a standard GRR data form.
  3. Calculate Repeatability (EV): For each sample+operator combination, calculate the range of the 2 replicates. Average all ranges → R̄. EV = R̄ / d₂. For 2 replicates, d₂ = 1.128.
  4. Calculate Reproducibility (AV): For each operator, calculate their grand average across all measurements. Range of operator averages → R_op. AV = √[(R_op / d₂*)² − (EV²/nr)] where d₂* depends on number of operators.
  5. Calculate %GRR: GRR = √(EV² + AV²). %GRR = GRR / TV × 100 (TV = total variation from the study).
  6. Calculate ndc: ndc = 1.41 × (σ_part / σ_GRR). Must be ≥5 for the gauge to distinguish between different product batches.
  7. Decision: If %GRR <10% and ndc ≥5 → approve the measurement system. If marginal (10–30%), document the risk. If >30% → investigate whether it is the equipment (EV high) or the operators (AV high) and address accordingly.
🧪 Results Example — Juice Brix Refractometer:

Operator 1 avg: 11.51 °Bx | Operator 2 avg: 11.54 °Bx
Repeatability σ (EV): 0.024 °Bx
Reproducibility σ (AV): 0.018 °Bx
GRR σ: √(0.024² + 0.018²) = 0.030 °Bx
Total study σ: 0.156 °Bx
%GRR = 0.030/0.156 × 100 = 19.2% → Marginal
ndc = 1.41 × 0.153/0.030 = 7.2 → OK (≥5)

Conclusion: ndc is OK (can distinguish batches), but %GRR is marginal. Since AV > EV, the issue is operator technique — one analyst reads slightly higher. Action: re-train both analysts on consistent sample prep (temperature equilibration) and re-run study.
💡
EV > AV: Equipment problem — calibrate, service, or replace the gauge.
AV > EV: Operator problem — re-train, clarify procedure, improve lighting/ergonomics.
📋 MSA for Audit and Lab — Beyond Gauge R&R
Attribute MSA (Pass/Fail)

When the measurement is a judgement (pass/fail, acceptable/reject) rather than a number. Used in visual inspection, sensory checks, audit checklists.

Method: Kappa statistic (Cohen's Kappa) κ = (P_observed − P_expected) / (1 − P_expected) κ > 0.9 → Excellent agreement κ 0.7–0.9 → Good κ 0.5–0.7 → Moderate — training needed κ < 0.5 → Poor — redefine criteria
🍶 Example — Bottle Cosmetic Inspection: 3 inspectors each evaluate 50 bottles (mix of good and defective) twice. Kappa = 0.72 (Good). Inspectors mostly agree, but some disagreement on "light scratches". Action: create visual standard reference board with borderline examples pinned at the workstation.
Linearity & Bias Study

Checks whether the gauge reads accurately across the full operating range (linearity) and whether it consistently reads high or low vs a reference value (bias).

Bias = mean(observed) − reference value % Bias = Bias / Process variation × 100 Target: |% Bias| < 5% Linearity: plot Bias vs Reference Value If bias changes across the range → linearity error (e.g., refractometer reads accurately at 8°Bx but reads +0.2°Bx too high at 14°Bx)
🧪 Example — pH Meter Linearity: Check meter at pH 3.0, 3.5, 4.0, 4.5, 5.0 using certified buffer solutions. If bias at pH 3.0 is +0.05 but at pH 5.0 is −0.08 → linearity problem → electrode aging → replace electrode. Critical for preservative efficacy monitoring where pH 3.8 vs 4.0 matters for legal compliance.
Hypothesis Testing
Using data to make statistically sound decisions — replacing opinion with evidence. The engine of the Analyse phase.
🧠 The Logic of Hypothesis Testing
H₀ (Null hypothesis): "No difference exists" / "No effect exists" H₁ (Alternative hypothesis): "A difference exists" p-value: probability of observing our data (or more extreme) IF H₀ is true Decision rule: p < 0.05 → Reject H₀ (statistically significant — evidence of a difference) p ≥ 0.05 → Fail to reject H₀ (insufficient evidence for a difference) α = significance level (risk of false positive) = 0.05 (default) β = probability of false negative (missing a real difference) Power = 1 − β (aim for ≥80%)
H₀ is TrueH₀ is False
Reject H₀Type I Error (α) — False alarmCorrect decision ✅
Fail to reject H₀Correct decision ✅Type II Error (β) — Missed signal
💡
Power and sample size: Before running a test, always ask: if a real difference exists, how likely am I to detect it? Use a power analysis to determine minimum sample size. Low n = underpowered study = missing real effects. In beverage testing, a practical rule: collect at least n=30 per group for continuous data before doing a t-test.
⚠️ Common Mistakes in Hypothesis Testing
  • ✗Confusing statistical vs practical significance. p=0.001 (very significant) with a mean difference of 0.01 mL fill volume is statistically real but operationally irrelevant.
  • ✗Skipping normality check. Applying t-test to sensory scores (ordinal data, not normal) → invalid result. Use Mann-Whitney instead.
  • ✗Insufficient sample size. Testing 5 samples per group with a t-test → power <30% — likely to miss real differences.
  • ✗Multiple testing without correction. Running 20 t-tests on the same data → expect 1 false positive by chance alone.
  • ✗Using paired test for independent samples or vice versa → biased results.
1️⃣ 1-Sample t-Test — Is the mean equal to a target?
H₀: μ = μ₀ (population mean equals target value) H₁: μ ≠ μ₀ (two-tailed) or μ > μ₀ / μ < μ₀ (one-tailed) t = (X̄ − μ₀) / (s / √n) Degrees of freedom: df = n − 1 Compare t to t-table (df, α) or use p-value directly. Assumptions: data approximately normal, random sample. Robust to non-normality for n ≥ 30 (Central Limit Theorem).

When to use

  • Comparing process mean to a target or spec
  • Checking if a new batch meets nominal
  • Lab qualification: does instrument read certified reference correctly?

What you get

  • t-statistic and p-value
  • Confidence interval for the true mean
  • Decision: is the mean significantly different from target?

Beverage use cases

  • Is new supplier sugar at 99.8% purity (target)?
  • Is this batch alcohol at 5.0% ABV as declared?
  • Is our calibrated scale reading 200g reference weight correctly?
🍺 Example — Alcohol Content Compliance:
A brewery must declare 5.0% ABV on their pale ale. The regulator will fine them if the actual content is significantly below 5.0%. QC samples n=15 bottles from a batch.
Results: X̄ = 4.87%, s = 0.18%.
H₀: μ = 5.0%, H₁: μ < 5.0% (one-tailed).
t = (4.87 − 5.0) / (0.18/√15) = −0.13/0.0465 = −2.80
p = 0.007 (df=14, one-tailed) → p < 0.05 → Reject H₀.
Conclusion: Batch mean is significantly below declared 5.0%. Hold batch for re-test / reformulation before shipping.
1-Sample t-Test — Alcohol Content vs 5.0% Target
2️⃣ 2-Sample t-Test — Are two group means different?
H₀: μ₁ = μ₂ (two groups have the same mean) H₁: μ₁ ≠ μ₂ (two-tailed) Welch's t-test (assumes unequal variances — recommended default): t = (X̄₁ − X̄₂) / √(s₁²/n₁ + s₂²/n₂) First test equality of variances: F-test: F = s₁²/s₂² (df1=n₁-1, df2=n₂-1) p > 0.05 → equal variances → use pooled t-test p < 0.05 → unequal → use Welch's t-test (safer default) Assumptions: both groups approximately normal, independent samples.
🧃 Example — Two Filling Lines Comparison:
Line A and Line B fill 330mL juice cartons. QC suspects Line B overfills. Data from one shift (n=40 each):
Line A: X̄=331.2 mL, s=1.8 mL
Line B: X̄=333.7 mL, s=2.1 mL
H₀: μ_A = μ_B, H₁: μ_A ≠ μ_B
Welch's t = (331.2−333.7)/√(1.8²/40 + 2.1²/40) = −2.5/√(0.081+0.110) = −2.5/0.437 = −5.72
p < 0.0001 → Highly significant.
Conclusion: Line B fills significantly more than Line A. Investigate Line B fill head calibration. Over-fill costs estimated at $18K/year in product giveaway.
💡
Paired vs Independent: Use a paired t-test when the same units are measured before and after (e.g., same bottles tested before and after a filter change). Use independent 2-sample t-test when comparing two completely separate groups. Pairing removes individual unit-to-unit variation and increases power dramatically.
🧪 Paired Example — Before/After Filter: 20 batches of beer measured for turbidity before and after installing a new filter cartridge. Since it's the SAME batches, use paired t-test. Result: mean reduction = 0.42 NTU, t = 4.88, p < 0.001 → filter significantly improves clarity.

3️⃣ One-Way ANOVA — Comparing 3 or more groups
H₀: μ₁ = μ₂ = μ₃ = ... = μk (all group means equal) H₁: at least one mean is different F = MS_between / MS_within = (SS_between / df_between) / (SS_within / df_within) df_between = k − 1 (k = number of groups) df_within = N − k (N = total observations) p < 0.05 → reject H₀ → at least one group is different (ANOVA does NOT tell you WHICH groups differ → use post-hoc tests) Post-hoc tests (after significant ANOVA): Tukey HSD: compares all pairs, controls family-wise error rate Bonferroni: more conservative, good for small number of comparisons

When to use

  • Comparing 3+ shifts, operators, lines, suppliers
  • Testing 3+ formulation batches
  • Multi-level factor screening

Assumptions

  • Each group approximately normal
  • Equal variances (Levene's test: p>0.05)
  • Independent observations

Beverage use cases

  • Do 3 shifts have the same Brix variation?
  • Does yeast strain (A, B, C) affect final ABV?
  • Do 4 sugar suppliers give the same product taste score?
🍺 Example — 3-Shift Brix Comparison:
Plant runs 3 shifts. QC suspects shift 3 has higher Brix variation. One month's data: n=28 measurements per shift.
Shift 1: X̄=11.48, s=0.12 | Shift 2: X̄=11.51, s=0.13 | Shift 3: X̄=11.67, s=0.19
One-way ANOVA: F=14.3, p<0.001 → Reject H₀ → at least one shift different.
Tukey HSD: Shift 3 vs 1 (p<0.001) ✓ different; Shift 3 vs 2 (p<0.001) ✓ different; Shift 1 vs 2 (p=0.31) — no difference.
Conclusion: Shift 3 runs systematically higher Brix. Investigate: different syrup pump setting, different operator mixing technique, different raw material batch used on night shift.
ANOVA — Brix by Shift (Shift 3 is different)
4️⃣ Chi-Square Test — Categorical data & proportions
Chi-square test of independence: H₀: Two categorical variables are independent (no association) H₁: There is an association between them χ² = Σ [(O_ij − E_ij)² / E_ij] E_ij = (row total_i × column total_j) / grand total df = (rows−1) × (columns−1) Rule: all expected cell counts ≥ 5 (if not, use Fisher's exact test) Chi-square goodness of fit: H₀: Observed frequencies match expected distribution H₁: They do not match df = k − 1 (k = number of categories)

When to use

  • Association between two categorical variables
  • Defect type distribution by machine/line
  • Audit findings by supplier vs category

What you get

  • χ² statistic and p-value
  • Contingency table analysis
  • Which cells contribute most to the association

Beverage use cases

  • Is defect type (fill/label/cap) independent of which line?
  • Are audit non-conformances associated with shift?
  • Are customer complaint categories independent of SKU?

Regression & Correlation
Quantifying relationships between variables — understanding what drives your process outputs (Y = f(X)). The mathematical backbone of root cause verification.
📈 Correlation — Is there a relationship?
Pearson r (for normal, continuous data): r = Σ[(xi−X̄)(yi−Ȳ)] / √[Σ(xi−X̄)² × Σ(yi−Ȳ)²] Range: −1 to +1 r = +1: perfect positive correlation r = −1: perfect negative correlation r = 0: no linear correlation Interpretation: |r| < 0.3 → weak / negligible |r| 0.3–0.7 → moderate |r| > 0.7 → strong Test significance: t = r√(n−2)/√(1−r²), df = n−2 p < 0.05 → correlation is statistically significant Spearman ρ: use when data is ordinal or non-normal (rank-based)
⚠️
Correlation ≠ Causation. Even a strong r=0.92 between two variables does not mean one causes the other. Example: beer sales and ice cream sales both correlate with temperature. In Six Sigma, use correlation to identify candidates, then use controlled experiments (DOE) to prove causation.
🍺 Example — Temperature vs Carbonation: You measure tank temperature and dissolved CO₂ on 35 consecutive batches. r = −0.88, p < 0.001. Strong negative correlation: as temperature rises, carbonation drops. This validates temperature as a key input X to investigate in your DMAIC Analyse phase. Follow up with a DOE to confirm the causal relationship.
Scatter Plot — Tank Temperature vs CO₂ Volume (r = −0.88)
📉 Simple Linear Regression
Ŷ = β₀ + β₁X + ε β₁ = Σ[(xi−X̄)(yi−Ȳ)] / Σ(xi−X̄)² ← slope β₀ = Ȳ − β₁X̄ ← intercept R² = SS_regression / SS_total = r² = % of variation in Y explained by X Interpret p-value of β₁: p < 0.05 → X is a significant predictor of Y Residuals must be: 1. Normal (histogram or Anderson-Darling test) 2. Constant variance (no fan shape in residuals vs fitted) 3. Independent (no pattern vs order)
🥤 Example — Syrup Temperature vs Brix Variation:
You suspect warm syrup causes high Brix variability. Data: 40 blends, X = syrup inlet temperature (°C), Y = final Brix deviation from target (°Bx absolute).
Regression output: Ŷ = 0.04 + 0.012×X. β₁ p-value = 0.001. R² = 61%.
Interpretation: For every 1°C increase in syrup temperature, Brix deviation increases by 0.012°Bx on average. Temperature explains 61% of the variation.
Prediction: at 25°C, expected deviation = 0.04 + 0.012×25 = 0.34°Bx.
Action: Install syrup temperature control set at ≤18°C. Predicted Brix deviation: 0.04 + 0.012×18 = 0.26°Bx (24% improvement in expected deviation, based on model).
Simple Regression — Syrup Temperature vs Brix Deviation
📊 Multiple Regression — Several X's predicting Y
Ŷ = β₀ + β₁X₁ + β₂X₂ + β₃X₃ + ... + ε Adjusted R² = 1 − [(1−R²)(n−1)/(n−k−1)] ← penalises for adding predictors; use this (not R²) to compare models VIF (Variance Inflation Factor): VIF > 5 → potential multicollinearity (predictors correlated with each other) VIF > 10 → serious multicollinearity → remove one of the correlated predictors p-value for each β_i: is this predictor significant after controlling for others?
🔑
Model building order: (1) Run correlation matrix to identify candidate X's. (2) Build model with all candidates. (3) Remove predictors with p > 0.05 and VIF > 5 one at a time. (4) Check residuals. (5) Validate on hold-out data if possible. In beverage, always include the most operationally important X's even if borderline significant.
🍺 Example — Dissolved Oxygen in Canned Beer:
Response Y = dissolved oxygen (ppb) in can headspace.
Candidate predictors: X₁ = fill speed (cpm), X₂ = seaming gap (mm), X₃ = CO₂ purge duration (s), X₄ = product temperature (°C).
Final model: Ŷ = 145 − 0.8×X₁ + 12×X₂ − 0.9×X₃ + 3.1×X₄
Adjusted R² = 78%. All predictors p < 0.05. VIF all < 3.
Most influential: seaming gap (β₂=12 ppb/mm). Action priority: tighten seaming gap specification from ±0.3 mm to ±0.1 mm. Predicted DO reduction from 85 ppb to 32 ppb. Impact: shelf life extended from 4 to 7 months.

Logistic regression example: Does fermentation temperature (X, continuous) predict whether a batch passes microbiological safety test (Y: pass=1, fail=0)? Logistic regression gives P(pass) = 1/(1+e^(−(β₀+β₁×temp))). If β₁ is negative and significant → higher temp → higher probability of failing the micro test.
Design of Experiments
The most powerful tool for discovering what really drives your process. DOE tests multiple factors simultaneously, revealing interactions that one-factor-at-a-time testing misses completely.
🧠 Why DOE — OFAT vs Factorial

One Factor At a Time (OFAT) — the traditional engineering approach — changes one X while holding everything else fixed. This misses interaction effects: situations where the effect of X₁ on Y depends on the level of X₂. In beverage processes, interactions are common and important.

⚠️
Classic OFAT failure: An engineer optimises CO₂ pressure at low temperature and gets 2.8 vol CO₂. Then optimises temperature at that CO₂ pressure. But the true optimum (2.4 vol CO₂ at high temperature) was never tested because the interaction between the two was never explored. A 2² factorial DOE would have found this in 4 runs.
2-factor full factorial: 2² = 4 runs (explore both factors + their interaction) 3-factor full factorial: 2³ = 8 runs 4-factor full factorial: 2⁴ = 16 runs 5-factor full factorial: 2⁵ = 32 runs → consider fractional Key outputs: Main effect of A: average change in Y when A goes low → high Main effect of B: average change in Y when B goes low → high Interaction AB: effect of A depends on level of B (and vice versa)
Interaction Plot — CO₂ Pressure × Temperature on Carbonation
Parallel lines = no interaction. Crossing/non-parallel lines = interaction present. Here, the optimal CO₂ pressure DEPENDS on temperature — OFAT would have missed this.

  3. Calculate main effect of B (temperature): B+ avg = 2.408, B− avg = 2.563. Main effect B = −0.155. Higher temp → less carbonation.
  4. Calculate main effect of C: +0.005 (negligible).

✅
DOE Golden Rules:
1. Always randomise run order.
2. Always include replicate runs (run the design 2–3 times for error estimation).
3. Run at genuine extremes (±20% of normal range) for maximum information.
4. Always confirm with a verification run at optimal settings before implementing.
5. Block on nuisance factors (e.g., if you must run across 2 days, use Day as a block).

Sampling & Auditing
How much to sample? How to structure an audit? Statistical sampling theory applied to beverage QC, incoming goods inspection, and internal audits.
🎯 AQL Sampling Plans — Incoming Goods Inspection

AQL (Acceptable Quality Level) is the maximum percentage of defectives considered acceptable in a lot. ISO 2859-1 (MIL-STD-105E) sampling tables give you the correct sample size and acceptance/rejection numbers for each lot size and AQL level.

AQL common levels in beverage industry: Critical defects (food safety, leakage): AQL 0.065 or 0.10 Major defects (fill volume, seam, Brix): AQL 0.65 or 1.0 Minor defects (cosmetic, label position): AQL 2.5 or 4.0 Normal inspection, Single sampling (ISO 2859-1 Table II-A): Lot size 1,200–3,200 → Code Letter K AQL 1.0 (Major) → n=125, Ac=3, Re=4 Meaning: sample 125 units; accept if ≤3 defects; reject if ≥4. Switching rules: Tightened inspection: 2 lots rejected in last 5 → tighten (smaller AQL) Reduced inspection: 10 consecutive lots accepted → reduce sample size
🍶 Incoming Inspection Example — PET Bottles from Supplier:
Delivery: 2,500 PET 500mL bottles. Critical defect = visible crack (AQL 0.065). Major defect = height OOT (AQL 1.0). Minor = cosmetic scratch (AQL 4.0).
Per ISO 2859-1, Lot Size 2,500, General Level II → Code Letter K.
Critical: n=125, Ac=0, Re=1 (zero tolerance for cracks)
Major: n=125, Ac=3, Re=4
Minor: n=200, Ac=14, Re=15
Sample 125 bottles: 0 cracks (pass), 2 height defects (pass), 8 scratches (pass). Accept delivery. Log results in supplier scorecard.
OC Curve — AQL 1.0, n=125 (Prob. Accept vs % Defective)
The OC curve shows the probability of accepting a lot as a function of the true % defective. At 1% defective (AQL), acceptance probability is ~95%. At 3% defective, probability drops sharply — this is the discrimination power of the plan.
📋 Sample Size Formulas for QC Decisions
Sample size for estimating a mean (continuous data): n = (Zα/2 × σ / E)² where E = maximum acceptable error from true mean σ = estimated process std dev Zα/2 = 1.96 for 95% confidence Sample size for detecting a difference (2-sample t-test): n per group = 2 × (Zα/2 + Zβ)² × σ² / δ² where δ = minimum detectable difference σ = pooled std dev Zβ = 0.842 for 80% power (β=0.20) 1.282 for 90% power Sample size for proportion (p-chart baseline): n = Z² × p(1−p) / E² Minimum: n such that np ≥ 5
💡
Rule of thumb for quick calculations:
• Continuous data, estimating mean: n≥30 for basic statistics
• Detecting a 1σ shift with 80% power: n≈17 per group
• Detecting a 0.5σ shift with 80% power: n≈64 per group
• Establishing control chart (Xbar-R): minimum 25 subgroups
• Capability study: minimum 30 subgroups (preferably 25–30 × n=5 = 125–150 individual readings)
🧪 Lab Sample Size — Detecting Brix Difference:
You want to detect if a new syrup supplier's product changes your Brix by 0.2°Bx or more. Historical process σ = 0.15°Bx. Target: 95% confidence, 80% power.
n = 2 × (1.96 + 0.842)² × 0.15² / 0.2²
n = 2 × (2.802)² × 0.0225 / 0.04
n = 2 × 7.85 × 0.5625 = 8.8 → round up to n = 9 per supplier.

Conclusion: Collect 9 batches from each supplier, run 2-sample t-test. Very feasible for a QC lab study.
🍺 Audit Sample Size — Seam Inspection:
Incoming batch of 10,000 cans. You want to estimate true seam defect rate within ±0.5% with 95% confidence. Expected rate p̂ ≈ 2% from historical data.
n = 1.96² × 0.02 × 0.98 / 0.005² = 3.84 × 0.0196 / 0.000025 = 3,004 cans.
That's expensive — instead, use AQL sampling (n=200, Ac=10 at AQL 4.0) which gives a practical decision rule rather than a precise estimate.

1️⃣ Mann-Whitney U Test — The Non-Parametric 2-Sample Test
H₀: The two populations have the same distribution (same median) H₁: The distributions differ (one tends to give larger values) Method (simplified): 1. Combine all observations from both groups, rank them 1 to N 2. For each group, sum the ranks: W₁ and W₂ 3. U₁ = n₁×n₂ + n₁(n₁+1)/2 − W₁ U₂ = n₁×n₂ + n₂(n₂+1)/2 − W₂ 4. U = min(U₁, U₂) 5. Compare to critical U table (or use p-value from software) For large samples (n₁, n₂ > 8): normal approximation with Z-score.
🍷 Example — Consumer Liking Scores for 2 Wine Label Designs:
30 consumers each rate ONE label design (independent groups, n=15 per group). Liking score 1–10 (ordinal, NOT normal).
Design A scores: 5,7,8,6,7,8,9,6,7,8,5,7,8,9,7 → Median = 7
Design B scores: 4,5,6,5,7,5,6,4,6,5,7,5,6,4,5 → Median = 5
Mann-Whitney U: U = 31, n₁=n₂=15. Critical U (α=0.05) = 72.
U < U_critical → Reject H₀. Design A is significantly preferred.
Note: A t-test would be inappropriate here because scores are ordinal (integer 1–10) and are not normally distributed.
Box Plot — Consumer Scores by Label Design (Mann-Whitney)
2️⃣ Kruskal-Wallis — Non-Parametric ANOVA (3+ groups)
H₀: All k groups come from the same distribution H₁: At least one group has a different distribution H = [12 / (N(N+1))] × Σ[Rⱼ² / nⱼ] − 3(N+1) N = total observations Rⱼ = sum of ranks in group j nⱼ = sample size of group j H ~ χ²(df = k−1) for large samples Post-hoc after significant KW: Use Dunn's test with Bonferroni correction to identify which pairs differ.
🍺 Example — Foam Stability by 4 Hop Varieties:
A craft brewery tests 4 hop varieties (A, B, C, D) on foam stability score (1–10, rated by trained panel). n=8 batches per hop variety. Scores are ordinal ratings, not normally distributed.
Kruskal-Wallis H = 14.2, df=3, p = 0.003 → At least one hop variety gives significantly different foam stability.
Dunn's post-hoc: Variety C vs A (p=0.001) significant; Variety C vs D (p=0.004) significant; other comparisons not significant.
Conclusion: Hop variety C gives consistently higher foam stability scores. Recommend switching to variety C for premium lines. Estimated additional material cost: +$0.04/litre, justified by premium quality positioning.
